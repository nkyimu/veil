# Veil — Day 0: Project Setup

**Date:** 2026-03-14
**Participants:** Amantu (human) + Granville (AI agent, Claude Opus 4.6)

## What Happened

### Human Direction
- Defined project concept: sovereign data vault where your agent guards data and charges for queries
- Chose "Agents that keep secrets" track for Synthesis Hackathon
- Set technical constraints: Base chain, USDC, Locus API, Self Protocol

### Agent Contribution
- Created GitHub repo and project structure
- Designed and implemented VeilVault.sol with:
  - Credential commitment storage (hash-based privacy)
  - Query payment system (pay-per-query in USDC)
  - Revenue distribution to data owners
  - Agent authorization for automated query answers
  - Platform fee mechanism
  - Query expiry and refund system
- Created agent/guardian.ts scaffold
- Set up Foundry with OpenZeppelin dependencies
- Verified contract compiles (`forge build` passes)

### Decisions Made
1. Credential commitments stored as keccak256 hashes — simple, effective privacy
2. Query answers are YES/NO only — selective disclosure without full data exposure
3. Agent can answer on behalf of data owner — autonomous monetization
4. Query expiry with auto-refund — prevents locked funds
5. Platform fee capped at 10% — fair value capture
